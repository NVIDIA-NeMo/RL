# vLLM 0.24 Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade NeMo-RL's vLLM generation extra from vLLM 0.20.0 to vLLM 0.24.0 while preserving synchronous and asynchronous rollout behavior.

**Architecture:** Keep the existing NeMo-RL worker and actor boundaries. First make the dependency graph internally consistent, then test the vLLM-facing import surface against an installed 0.24 wheel, and patch only APIs that demonstrably moved. Validate CPU/static behavior locally and GPU engine startup in the standard nightly container on GB200.

**Tech Stack:** Python 3.13.13, uv, PyTorch 2.11.0/CUDA 13.0, vLLM 0.24.0, FlashInfer 0.6.12, pytest.

## Global Constraints

- Base all work on NeMo-RL `origin/main` commit `1c82fdef20d67aeaadc3239ca4c4956fe5d1ff8a`.
- Keep `torch==2.11.0`, `torchvision==0.26.0`, and Python `>=3.13.13,<3.14`.
- Use official vLLM 0.24.0 Linux wheels for both `aarch64` and `x86_64`.
- Do not change policy-training, reward, logprob, or sampling semantics.
- GPU validation must use CUDA Graph enabled unless a test explicitly targets eager mode.
